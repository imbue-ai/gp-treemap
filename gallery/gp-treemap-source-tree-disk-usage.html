gp-treemap — disk usage

Cell area is bytes on disk. Cells are colored by the color mode above. Each folder's pixels are subdivided among its children, so the shape of the tree is visible in the layout.

Mouse

  • Hover: see full path + size in the tooltip.
  • Click a cell: select it; the breadcrumb lights up.
  • Scroll wheel while a cell is selected: move the highlight up or down the ancestor chain.
  • Double-click a cell: zoom into it. Double-click a parent in the breadcrumb to zoom back to that ancestor.

Breadcrumb (the path row)

  • Click a segment to focus that ancestor.
  • Double-click to zoom to it. The home icon on the left jumps to the tree root.

Component toolbar

  • Depth: how many levels below the current zoom to draw. +/−, or type a number. Snaps to ∞ once you reach the tree's deepest level.
  • Labels: show each leaf's full path from the zoom root inside its cell, when it fits.

URL

All state (zoom / focus / theme / palette / depth …) is serialized into the URL hash as a single JSON blob, so you can copy a link to any view.
